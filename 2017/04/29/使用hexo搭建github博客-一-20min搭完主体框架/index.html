使用Hexo搭建Github博客：(一)20min搭完主体框架

4月 29, 2017

因为本文是对学习过程的记录和总结，大量借鉴了20分钟教你使用hexo搭建github博客，且对遇到的坑做了修改和补充，介绍比较详细，所以适合新手阅读。

由于使用的是苹果机，所以本教程以Mac OS为参考，其实mac与windows搭建github博客相差无几，如果有用windows的小伙伴感到不适，可以参考手把手教你建github技术博客教程。

背景

为什么开始博客？

作为一个刚接触IT行业的新人，面对各种计算机的疑难杂症，一步三搜索，每天Baidu，Google是必不可少的。

IT行业日新月异，幸运的是，有很多先辈通过博客、视频、MOOC等形式分享自己的经验，不幸的是，海量资料却有极其强的时效性。经常需要进行筛选、组合来完成自己当下的目标。（这个很机器学习）

所以，作为后辈，

  • 学而时习，学而时思：有必要把学到的知识记录整理，促进记忆理解，
  • 受人恩惠：有责任把自己踩过的坑、整合的最新知识记录下来，供朋友们参考。

所以打算用这个文章系列和大家分享使用hexo搭建github博客的过程。

为什么是Hexo和Github？

起初仅有一个写博客的想法，于是围绕这个想法做了简单的调研，除了查找中文IT资料经常见到的新浪、网易、天涯、CSDN、简书、博客园等博客平台以及各种个人网站，也陆续了解了wordpress，hexo等等创建个人网站或博客的工具。

作为一名颜值控，我对博客的要求不高，只有5个：颜值高、排版好、简洁风、无广告、方便写，于是，迅速pass掉了新浪、CSDN等造成严重视觉污染的方案，现有中文博客平台也就简书还凑活。

然而，这还不够，了解了几个hexo和wordpress的漂亮模板，最后将目标集中在了hexo和wordpress。这两个都可以自建平台，

考虑到自己

  • 具有一定的编程功底（了解Github，熟悉命令行）

  • 没有服务器，暂时不打算申请域名

  • 对百度能不能搜到我的博客不着急（有方法，但麻烦）

  • Hexo又趋向简洁风

所以最终选择了Hexo+GitHub搭建个人博客的方式，下面我们正式开始探索这个新奇的小世界。

途中可以看出github的仓库有两个地址，HTTPS的和SSH的，以这里的Blogname仓库为例，他的两个地址分别是：

错误：Deployer not found: git
​ 方法1：可能是没安装hexo-deployer-git，可在终端执行命令：npm install hexo-deployer-git --save，然后继续执行hexo deploy指令进行部署。

​ 方法2：如果方法1没效果，可能是错误添加了plugin，需要把Blogname/_config.yml文件中你新添加的plugin内容删掉。
